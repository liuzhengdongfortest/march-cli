What March does

A coding agent designed around fresh facts.

Instead of treating chat history as the source of truth, March keeps stable instructions, session identity, recent turns, and memory hints in clear layers.

↻

Context reconstruction

Every model call is assembled from stable layers and current workspace facts, reducing stale assumptions and cache churn.

◈

Markdown memory

Memories live as Markdown files. Passive recall gives short hints; the agent opens full memory only when needed.

⌁

Real terminal tools

March can run commands, preserve interactive shells, inspect output, edit files, and verify changes without leaving the CLI.

▣

MCP ready

External MCP tools and instructions plug into the runtime without turning provider routing into prompt policy.

⌕

Web-aware work

Built-in web search and page fetch tools help March inspect current docs, APIs, and references when local facts are not enough.

✦

Terminal UI first

A focused TUI keeps agent output, tool activity, permissions, status, and shell panes close to the coding workflow.

Core idea

The prompt is assembled, not accumulated.

User turn
  ↓
March context assembly
  ├── system_core      stable rules
  ├── injections       MCP / extensions
  ├── session_identity cwd / platform
  └── recent_chat      turns + recall hints
  ↓
Model call
  ↓
Tools read current files, shell, memory, web
1

Stable first

Large, low-change prompt layers stay at the front to keep provider prefix cache useful.

2

Facts on demand

Files, terminal output, diagnostics, and memory bodies are read by tools when the task needs them.

3

Small recalls

Memory hints point to relevant knowledge without flooding every prompt with full notes.
